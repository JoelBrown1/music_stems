# PyInstaller Launcher Design Spec

**Date:** 2026-06-26
**Status:** Approved

## Summary

Package the stem-splitter PyQt6 app into a double-clickable `.app` bundle (Mac), `.exe` (Windows), and binary (Linux) using PyInstaller. Friends receive a zip with no Python or dependency setup required. All deps — including the ML stack (madmom, librosa, basic_pitch, sounddevice) — are bundled.

## New Files

All new files; zero changes to existing app code.

| File | Purpose |
|---|---|
| `StemSplitter.spec` | PyInstaller spec controlling bundling |
| `scripts/build.sh` | One-command build for Mac/Linux |
| `scripts/build.bat` | One-command build for Windows |
| `resources/icon.png` | 1024×1024 source icon (generated with PIL) |

`dist/` and `build/` added to `.gitignore`.

## Bundling Strategy

### Output format

Folder mode (`onedir`), not single-file. Single-file mode causes 15–30s startup on heavy ML stacks. Mac's `.app` bundle format hides the internal folder so users see a normal app icon. Windows gets `dist/StemSplitter/StemSplitter.exe` (zip the whole folder to share).

### Data files collected

PyInstaller's `collect_data_files()` is used for:
- `librosa` — kaiser filter tables
- `basic_pitch` — ML model weights
- `madmom` — pretrained models

### Hidden imports

Declared explicitly in the spec for modules PyInstaller's static analysis misses:
- `sounddevice`, `soundfile`
- `resampy`
- `scipy.signal`, `scipy.fft`
- `madmom.features`, `madmom.processors`
- `librosa.core`, `librosa.beat`
- `sklearn` (imported transitively by basic_pitch)

### App settings

- `--windowed` — no terminal window on launch
- Entry point: `stem_splitter/main.py`
- App name: `StemSplitter`
- Bundle ID: `com.joelbrown.stemsplitter` (Mac)
- Icon: `resources/icon.icns` (Mac), `resources/icon.ico` (Windows)

### Icon generation

A simple 1024×1024 PNG is generated by a Python script using PIL during the build (`scripts/make_icon.py`). It renders a dark background with a waveform/stem visual and the letter "S". The script also converts:
- PNG → `.icns` using macOS `iconutil` (Mac only)
- PNG → `.ico` using PIL (cross-platform)

## Build Workflow

### Mac / Linux

```bash
./scripts/build.sh
```

1. Activates `.venv`
2. Installs `pyinstaller` into `.venv` if not present
3. Runs `make_icon.py` to generate `resources/icon.icns` + `resources/icon.ico`
4. Runs `pyinstaller StemSplitter.spec --noconfirm`
5. Prints: `Built: dist/StemSplitter.app`

### Windows

```bat
scripts\build.bat
```

Same steps, using `.venv\Scripts\activate`.

### Sharing

```bash
cd dist && zip -r StemSplitter-mac.zip StemSplitter.app
```

Send via AirDrop, Google Drive, or any file sharing. On first launch macOS shows a Gatekeeper warning ("unidentified developer") — recipients right-click → Open once to bypass. This is expected for unsigned apps.

## Error Handling

**Hidden import missing at runtime:** app crashes with `ModuleNotFoundError`. Fix: add the module to `hiddenimports` in `StemSplitter.spec` and rebuild. Typically needs 1–2 iterations on first build.

**`iconutil` not available on Linux/Windows:** `make_icon.py` skips `.icns` generation and falls back to a PIL-generated `.ico` only. The spec references whichever icon file exists for the current platform.

## Out of Scope

- Code signing / notarization (removes Gatekeeper warning; requires $99/yr Apple Developer account)
- Auto-updates
- `.dmg` installer or `.msi` installer wizard
- Cross-compilation (Windows `.exe` must be built on Windows; Mac `.app` must be built on Mac)
